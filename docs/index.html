Expose with Cloudflare Tunnel

Get a permanent HTTPS URL with an access-key gate — free, no open ports, outbound-only.

1

Start the stack first

The tunnel exposes what's already running locally.

cd Project-X && ./setup.sh
2

Install cloudflared (one-time)

Requires Homebrew — a single lightweight binary.

brew install cloudflare/cloudflare/cloudflared
3

Deploy the auth-gate Worker (one-time)

Creates a permanent protected URL at project-x-auth.YOUR_NAME.workers.dev.

./scripts/deploy-worker.sh
4

Run the tunnel

Opens the tunnel, gets a URL, and auto-links the Worker to point at it.

./scripts/cloudflare-tunnel.sh
5

Share your permanent URL

Visitors enter your access key to reach the app. The direct tunnel URL stays private.

🔐 https://project-x-auth.YOUR_NAME.workers.dev
6

Change access key anytime

Takes effect instantly — no redeployment needed.

wrangler secret put ACCESS_KEY --config cloudflare-worker/wrangler.toml
How it works: The Cloudflare Worker sits at a permanent workers.dev URL and shows a login page. Once the correct key is entered a signed session cookie is set and all requests proxy through to the tunnel. cloudflared opens an outbound-only connection — no ports are exposed on your machine. Press Ctrl+C to stop the tunnel and revoke all access instantly.
Local Setup

Run locally in minutes

Docker Desktop is all you need — works on Mac, Windows, and Linux.

1

Clone the repository

Clone Project X with all components included.

git clone --recurse-submodules https://github.com/WassimOmran/Project-X.git cd Project-X
2

Configure your environment

Copy the example env and add at least one LLM API key.

cp .env.example .env # Edit .env → OPENAI_API_KEY=sk-proj-...
3

Launch everything

One script starts all 6 Project X services — document engine, chat, search, storage, and more.

./setup.sh
4

Connect the components

Open the Document Engine, register, go to Settings → API Keys → create a key, add to .env.

# .env RAGFLOW_API_KEY=ragflow-xxxx docker compose restart kotaemon
5

Start chatting

Login (admin / admin), upload documents, and ask questions with inline citations.

Project X Chat → http://localhost:7860 Document Engine → http://localhost:80
