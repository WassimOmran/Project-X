Expose Project X with Cloudflare Tunnel

Run the stack on your Mac and get a live public HTTPS URL instantly — no server, no open ports, completely safe

1

Make sure the stack is running first

The tunnel just exposes what's already running locally. Start the stack if you haven't yet.

cd Project-X ./setup.sh
2

Install cloudflared (one-time)

Requires Homebrew. This installs the Cloudflare tunnel client — a single lightweight binary.

brew install cloudflare/cloudflare/cloudflared
3

Run the tunnel script

This single script opens two tunnels simultaneously — one for Kotaemon chat UI and one for RAGFlow admin — then prints both public URLs.

./scripts/cloudflare-tunnel.sh
4

Share your public URLs

The script prints two https://xxxx.trycloudflare.com URLs. Share the Kotaemon URL with anyone — they can chat with your RAG from anywhere in the world.

🧠 Kotaemon Chat UI → https://xxxx.trycloudflare.com ⚡ RAGFlow Admin UI → https://yyyy.trycloudflare.com
5

Stop the tunnel anytime

Press Ctrl+C in the terminal running the tunnel script. Both tunnels close instantly and your Mac goes back to being completely private. URLs expire when the tunnel stops.

How it works (and why it's safe):
cloudflared opens an outbound-only connection to Cloudflare — like a browser opening a website. No ports are opened on your router. No inbound traffic reaches your Mac directly. All requests flow through Cloudflare's servers first. You can stop everything with Ctrl+C.
Local Setup

Run Locally in Minutes

Docker Desktop is all you need — works on Mac, Windows, and Linux

1

Clone the repository

Clone Project X with all submodules (RAGFlow + Kotaemon included).

git clone --recurse-submodules https://github.com/WassimOmran/Project-X.git cd Project-X
2

Configure your environment

Copy the example env and add at least one LLM API key (OpenAI, Anthropic, Google, or use Ollama for free).

cp .env.example .env # Edit .env and add: OPENAI_API_KEY=sk-proj-...
3

Launch everything

One script starts all services: RAGFlow, Kotaemon, Elasticsearch, MySQL, MinIO and Redis.

./setup.sh
4

Connect RAGFlow API key to Kotaemon

Open RAGFlow, register, go to Settings → API Keys → create a key, paste into .env, then restart.

# In .env: RAGFLOW_API_KEY=ragflow-xxxxxxxxxxxxxxxxxxxxxxxx # Then restart Kotaemon: docker compose restart kotaemon
5

Upload documents and start chatting

Open Kotaemon at localhost:7860, login (admin/admin), upload files via RAGFlow Collection, and chat!

Kotaemon UI → http://localhost:7860 RAGFlow UI → http://localhost:80
